# BuildTools — 多边形建筑生成工具

## 概述

BuildTools 是一个 Unity Editor 工具，用于在场景中绘制多边形边界，并沿边界排列 Prefab 建筑。支持随机、等距、顺序三种排列模式，适用于快速搭建城市场景、园区布局、公交站周边建筑等。

```
└── Settings/
    └── BuildToolsSettings.asset  # 自动生成的持久化设置（ScriptableObject）
```

## 快速上手

### 1. 打开工具窗口

Unity 菜单 → **Window > Build Tools**

窗口顶部有一个 **「进入编辑」/「退出编辑」** 按钮：
- 点击进入后，工具开始拦截场景视图点击（Q/W 切换模式）
- 点击退出后，恢复 Unity 正常操作，不会再误触添加顶点

### 2. 两种工作模式

工具通过 Unity 原生工具栏切换，与 Unity 快捷键一致：

| 快捷键 | Unity 工具 | 对应模式 | 功能 |
|--------|-----------|----------|------|
| **Q** | View（手形） | **添加模式** | 点击地面添加顶点，顶点显示为绿色圆点 |
| **W** | Move（移动） | **编辑模式** | 顶点出现 XYZ 坐标轴，可沿轴拖拽移动 |
| E/R/T 等 | 其他工具 | 禁用 | 场景交互暂停 |

### 3. 添加模式（Q）— 绘制多边形

1. 按 **Q** 确认工具栏为手形图标
2. 在 **Scene View** 中**左键点击地面**，每次点击添加一个顶点
3. 连续顶点之间会自动绘制黄色边线
4. 最后一个顶点到鼠标位置会显示**虚线预览**
5. **Shift + 点击边**：在已有线段上插入一个新顶点（边会高亮为绿色）
6. 窗口状态栏显示 `🟢 添加模式（按 Q）`

### 4. 编辑模式（W）— 调整顶点

1. 按 **W**，工具栏会短暂显示移动图标后恢复手形（工具内部重置为 None 以避免冲突）
2. 每个顶点出现 **XYZ 坐标轴**（PositionHandle）
3. **点击并拖拽**坐标轴可沿轴精确移动顶点位置
4. 移动后 Y 坐标会自动投射回地面
5. 窗口状态栏显示 `🟡 编辑模式（按 W）`
6. 编辑模式下**点击地面不会添加新顶点**

### 5. 闭合多边形

按 **Enter** 键或点击窗口中的「闭合多边形」按钮。闭合后会出现**半透明斑马纹填充**。

### 6. 配置建筑 Prefab

在窗口的 **「建筑 Prefabs」** 区域：

- **批量拖拽**：从 Project 视图多选 Prefab 文件，拖入顶部的拖拽区域，自动为每个 Prefab 创建槽位
- 点击 **「+ 添加 Prefab 槽位」** 手动添加空槽位
- 拖入建筑 Prefab 到对象字段
- 调整**权重**（0-1），权重越高被选中的概率越大
- 可添加多个槽位，支持混合多种建筑

### 7. 生成建筑

生成建筑只需满足两个条件：
- 至少 **2 个顶点**（即至少 1 条边）
- 至少 1 个启用的、包含 Prefab 的槽位

点击 **「🎲 生成建筑」** 按钮，或开启 **「实时预览」** 开关。

开启实时预览后，调节任何参数都会自动重新生成建筑（0.3 秒防抖），无需手动点击生成按钮。

> **注意**：多边形不要求闭合即可生成建筑，建筑将沿已有边线排列。闭合多边形（≥3 顶点）可额外显示斑马纹填充。

## 窗口 UI 说明

### 多边形区域

- 显示所有顶点的世界坐标，可直接编辑数值
- 每个顶点可单独删除（点击 × 按钮）
- 「闭合多边形」/「打开多边形」— 切换闭合状态（需 ≥3 顶点）
- 「清空顶点」— 删除所有顶点
- 「撤销上一个」— 移除最后一个顶点

### 建筑 Prefabs 区域

- 每个槽位包含：Prefab 对象字段、权重滑块、启用开关、删除按钮
- 最多可添加任意数量的槽位

### 放置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| **排列模式** | 随机 / 等距 / 顺序 | 随机 |
| **间距** | 等距/顺序模式下模型边缘到边缘的空隙 | 0 |
| **建筑数量** | 沿边缘放置的建筑总数 | 10 |
| **随机种子** | 相同种子产生相同结果（仅随机模式） | 0 |
| **边缘偏移** | 正值 = 向多边形内部偏移，负值 = 向外部偏移 | 0 |
| **随机边缘偏移** | 在基础偏移上叠加随机 ±值 | 关闭 |
| **随机偏移范围** | 随机偏移的范围（±值，单位米） | 0 |
| **最小间距** | 建筑之间的最小距离（0 = 不检测） | 0 |
| **朝向模式** | 见下方说明 | 面向中心 |
| **世界方向** | 固定世界方向（仅 WorldDirection 模式） | (0, 0, 1) |
| **边缘旋转偏移** | 沿边缘方向额外旋转角度 ±180°（仅 AlongEdge 模式） | 0° |
| **边对齐** | 左对齐 / 居中 / 右对齐（仅 AlongEdge 模式） | 居中 |
| **随机旋转** | 在基朝向基础上添加随机旋转偏移 | 关闭 |
| **旋转范围** | 随机旋转的角度范围（±°） | 15° |
| **覆盖图层** | 为生成的建筑设置统一图层 | 关闭 |
| **分组建筑** | 将所有生成的建筑放在同一个父对象下 | 开启 |
| **分组名称** | 父 GameObject 的名称 | _Buildings |

#### 间距说明

| 排列模式 | 间距 = 0 时的行为 |
|----------|-------------------|
| **等距** | 自动按「建筑数量」等分周长（中心到中心） |
| **顺序** | 模型边缘紧挨（前一个前缘贴合后一个后缘） |
| **随机** | 间距参数不生效 |

#### 边对齐（仅 AlongEdge 模式）

| 对齐方式 | 说明 |
|----------|------|
| **居中** | 模型中心位于边缘线上，再叠加边缘偏移 |
| **左对齐** | 模型完全在边缘内侧（pivot + 半宽偏移） |
| **右对齐** | 模型完全在边缘外侧（pivot - 半宽偏移） |

### 朝向模式

| 模式 | 说明 |
|------|------|
| **FacePolygonCenter** | 建筑前方朝向多边形内部中心 |
| **AlongEdge** | 建筑前方沿边缘切线方向。直线段 = 边方向；弧段 = 采样点的弧切向量（沿弧动态变化），可叠加旋转偏移和对齐方式 |
| **WorldDirection** | 固定世界方向（可自定义） |
| **Random** | 完全随机 Y 轴旋转 |

> **圆角下的 AlongEdge**：启用圆角转折后，弧段上的建筑朝向会随弧曲线动态变化，每个建筑面向其所在弧段的切向，实现建筑沿弧线自然排列的效果。

### 可视化设置

- **轮廓颜色** — 多边形边线颜色（默认黄色）
- **填充颜色** — 斑马纹主色，第二种颜色自动由主色降低透明度生成
- **手柄颜色** — 顶点球形手柄颜色（默认绿色）
- **手柄大小** — 顶点手柄的屏幕尺寸
- **预览线颜色** — 未闭合时从最后顶点到鼠标的虚线颜色
- **圆角转折** — 启用后顶点处以圆弧替代尖角，多边形边线和建筑放置均使用圆角后的边线
- **圆角半径** — 圆弧半径（米），过大时自动按相邻边长的一半约束
- **射线检测图层** — 点击添加顶点时射线检测的图层

> **圆角转折说明**：启用后，多边形的每个顶点处会生成一段圆弧，边线由「直线段 → 弧段 → 直线段 → ...」交替组成。建筑也会沿这些弧段放置——在弧段上采样位置时，切线方向随弧线动态变化，使 `AlongEdge` 朝向模式的建筑能沿弧线自然旋转。

## 放置算法说明

### 边列表构建

1. 从多边形顶点构建边列表
2. **启用圆角转折时**：使用 `ComputeCornerTangents` 计算每个顶点的圆角切点，生成「直线段 → 弧段 → 直线段 → ...」交错的边列表
3. 每条边（直线或弧）包含：起终点、长度、方向/切线、内向法线、累计周长起始值
4. 弧段额外包含：弧心 (`arcCenter`)、起方向 (`arcFromDir`)、扫掠角度 (`arcAngleDegrees`)、半径 (`arcRadius`)

### 边上采样（统一直线/弧）

`GetPointOnEdge` 方法根据 `tNorm`（0~1）在边上采样，返回**位置**、**切线**、**法向**：
- **直线段**：位置 = `Lerp(start, end, t)`，切线 = 边方向，法向 = 内向法线
- **弧段**：位置 = 弧心 + 旋转后的方向 × 半径，切线 = 弧的切向量（逆时针 = XZ 旋转 90°），法向 = 指向弧心

### 随机模式 (Random)
1. 计算多边形每条边在 XZ 平面上的长度（弧段 = 弧长）
2. 按边长加权随机选择边（长边被选中的概率更高）
3. 在选中的边上随机采样位置（`GetPointOnEdge`）
4. 应用边缘偏移 + 计算朝向 + 选择 Prefab
5. 检测最小间距约束（不满足则重试，最多 10 次）

### 等距模式 (Equidistant)
1. 计算总周长（含弧段），按间距或数量确定步长
2. 随机起始偏移，沿周长均匀分布
3. 每个步长位置定位到对应边，通过 `GetPointOnEdge` 获取坐标、切线和朝向
4. 间距 > 0 时：步长 = 2×模型平均半宽 + 间距，自动计算可放置数量

### 顺序模式 (Sequential)
1. 从多边形第一个顶点（索引 0）出发
2. 按固定间距沿边缘方向依次放置
3. 每个建筑使用真实 Prefab 包围盒计算前后缘
4. 间距 = 0 时模型紧挨排列（前一个前缘贴合后一个后缘）
5. 跨边自动衔接（包括直线段和弧段），直到超出周长或达到建筑数量

## Undo 支持

所有操作均支持 **Ctrl+Z** 撤销：

- 添加 / 移动 / 删除顶点
- 切换闭合状态
- 修改任何参数
- 生成建筑（逐个注册）
- 清除建筑（逐个销毁）

## 技术细节

- **Unity 版本**: 2022.3+
- **渲染管线**: Built-in Render Pipeline
- **多边形数学**: 所有计算在 XZ 平面上进行，Y 坐标通过射线检测获取
- **边线类型**: `PolygonEdge` 支持直线段和弧段两种类型（`isArc` 区分）。弧段包含弧心、起方向、扫掠角度、半径等数据
- **统一样条采样**: `GetPointOnEdge` 对直线和弧段提供统一接口，返回位置、切线、内向法线
- **圆角计算共享**: `ComputeCornerTangents` 同时服务于可视化渲染（`DrawRoundedPolygonEdges`）和放置算法（`BuildEdgeList`），消除重复代码
- **填充渲染**: 使用自定义 ZebraFill Shader 通过 Mesh + `Graphics.DrawMeshNow` 绘制双色斑马纹，条纹沿世界空间 XZ 对角线方向
- **设置持久化**: ScriptableObject 存储在 `Assets/BuildTools/Settings/`
- **场景引用**: 通过隐藏的 `BuildToolsSceneSettings` GameObject 跟踪生成的建筑

## 注意事项

- 添加顶点前确保场景中有可射线检测的地面/碰撞体
- 如果没有命中碰撞体，顶点会自动投影到 **Y=0** 平面
- 生成建筑只需 ≥2 顶点（至少一条边），闭合不是必须的
- 至少需要一个启用的、包含 Prefab 的槽位才能生成
- 凹多边形填充显示可能有轻微视觉瑕疵，但不影响放置精度
